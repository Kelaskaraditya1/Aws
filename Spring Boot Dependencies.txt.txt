<----------------------------------------------------------------------------------------Circuit Breaker----------------------------------------------------------------------------------->


5) Circuit Breaker Pattern: using Resilience 4-j dependency.

Circuit breaker is used to stop gracefully, if any of the microservice fails to give response or is down.
It is used to implement fault-tolerance mechanism.

working:

1) It works as a Finite State Autometa which has 3 states open,close and half-open 
2) It works as a Electrical circuit
3) Initially when all services are active than the state is close, Circuit is closed.
4) When any of the service is down than it moves to open state and it remains in open state for some time, Circuit is open.
5) After some time service moves to half open state and again it hit's the service if it get's satisfactory response than it moves to close state else it moves to open state.
6) We have to add a fallback method means if the service is down that method will be executed we have to use @Circuitbreaker annotation and it takes 2 parametes name and fall back method. the name will be used in the application.properties and the method arguments in the fall back method should be same as the actual api call method, and the return type should also be the same.	

1) Add dependencies:

	1)actuator
	2)aop (add aop to actuator dependency)
	3)Resilience-4j:

		<dependency>
			<groupId>io.github.resilience4j</groupId>
			<artifactId>resilience4j-spring-boot2</artifactId>
			<version>2.0.2</version>
		</dependency>

2) application.properties:


# Actuator configurations:

management.health.circuitbreakers.enabled=true
management.endpoints.web.exposure.include=health
management.endpoint.health.show-details=always

#Resilience-4j configurations: can be multiple as no of @Circuitbreaker's added to restcontroller , change "ratingHotelBreaker" with the name mentioned in the annotation.

resilience4j.circuitbreaker.instances.ratingHotelBreaker.registerHealthIndicator=true
resilience4j.circuitbreaker.instances.ratingHotelBreaker.eventConsumerBufferSize=10
resilience4j.circuitbreaker.instances.ratingHotelBreaker.failureRateThreshold=50
resilience4j.circuitbreaker.instances.ratingHotelBreaker.minimumNumberOfCalls=5
resilience4j.circuitbreaker.instances.ratingHotelBreaker.automaticTransitionFromOpenToHalfOpenEnabled=true
resilience4j.circuitbreaker.instances.ratingHotelBreaker.waitDurationInOpenState=6s
resilience4j.circuitbreaker.instances.ratingHotelBreaker.permittedNumberOfCallsInHalfOpenState=3
resilience4j.circuitbreaker.instances.ratingHotelBreaker.slidingWindowSize=10
resilience4j.circuitbreaker.instances.ratingHotelBreaker.slidingWindowType=COUNT_BASED
