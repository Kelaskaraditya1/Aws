Aws Notes

1) IAM Service(Identity Access Management):

in Aws there is a root user, with the email with which the aws has been created than there is "IAM" user who is made by the root user it can be a user or a backend application which can access the resources of aws.
while creating the user we can specify in the policy section the permissions to services to which the user can access. 

2) S3 Bucket:

AWS SDK:

<dependency>
    <groupId>software.amazon.awssdk</groupId>
    <artifactId>s3</artifactId>
    <version>2.41.29</version>
</dependency>

configuration:

aws.s3.bucket-name=
aws.s3.region=
aws.s3.access-key=
aws.s3.secret-key=

3) Ec2 instance:

 1) while creating EC2 Instance:

    1) download the .pem file if creating new key pair and should be in the dir where the .jar is present 

    2) if creating a new security grp than add there should be following policies:

        1) type:ssh | port:22 : for remote connection with cmd
        2) type:CustomTcp | port:8080  | source:AnywhereIPV4

    3) while creating ec2 instance paste the below shell script for a welcome page when root ip is hit

    #!/bin/bash
    yum update -y
    yum install -y httpd
    systemctl start httpd
    systemctl enable httpd

    echo "<html>
    <head><title>My First AWS Page</title></head>
    <body>
        <h1>Hello from EC2!</h1>
        <p>This is a static HTML page hosted on Apache.</p>
    </body>
    </html>" > /var/www/html/index.html


 2) for ssh remote connection:

    1) create the .jar file of the spring boot application using the command: mvn clean package
    2) copy the .jar file in the dir where the .pem file is.
    3) connect to the ec2 instance:ssh -i <KEY_PAIR_FILE> ec2-user@<EC2_PUBLIC_DNS>
    4) install the enviornment for running the application(for eg java): sudo dnf install java-17-amazon-corretto -y
    5) now open other terminal for uploading the .jar file: scp -i <KEY_PAIR_FILE> <JAR_FILE> ec2-user@<EC2_PUBLIC_DNS>:~
    6) check whether the .jar is present and run it using: java -jar <name of the jar>
    7) to keep the instance running: nohup java -jar <JAR_FILE> > app.log 2>&1 &
